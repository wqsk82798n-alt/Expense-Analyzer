# 스마트 가계부

## 1. Business Purpose

현재 사회에는 다양한 가계부 앱이 존재하며 이를 통해 사용자는 자신의 소비 내역을 기록하고 확인할 수 있다. 이러한 앱은 소비 금액을 직관적으로 보여준다는 장점이 있지만 소비 증가의 원인이나 패턴에 대한 분석 기능은 부족한 한계가 있다.

따라서 스마트 가계부는 단순 소비 기록을 넘어 사용자의 소비 데이터를 분석해 지출 패턴과 증가 원인을 설명하고, 이를 기반으로 소비 개선을 유도하는 것을 목표로 한다.

스마트 가계부의 목적은 개인의 소비 데이터를 체계적으로 관리하고 분석하여 사용자의 재정 상태를 직관적으로 이해하고 개선하도록 돕는 것이다.

### 주요 기능 목표

- 지출 패턴 분석을 통한 소비 습관 개선 유도  
- 예산 대비 지출 관리 및 초과 방지  
- 데이터 기반 소비 인사이트 제공

## 3. Concept of Operation

회원가입 / 로그인	- 사용자 인증 수행
지출 데이터 입력 - 소비 데이터 입력
지출 데이터 수정 - 기존 데이터 수정
지출 데이터 삭제 - 기록 삭제
데이터 조회 - 기간별 소비 내역 조회
카테고리 분류 - 자동/수동 분류
지출 분석 - 소비 패턴 분석
예산 설정 - 지출 한도 설정
예산 초과 알림 - 초과 시 알림 제공
데이터 Import - 외부 데이터 자동 입력
데이터 Export - 파일 다운로드
통계 시각화 - 그래프 형태 제공
추천/피드백 제공 - 소비 패턴 및 원인 설명

1) Login / Register (로그인 및 회원가입)
• Purpose: 앱을 안전하게 사용하기 위해 사용자 신원을 확인하고 개인화된 데이터를 보호
• Approach: ID/PW 및 소셜 로그인 방식을 제공, 서버 DB와 대조하여 인증을 수행
  회원가입 시 중복 ID 체크 및 암호화 저장을 거침
• Dynamics: 앱 최초 실행 시 또는 로그아웃 상태에서 접근할 경우 발생
• Goals: 사용자 인증 시스템 구축 및 개인별 독립된 데이터 환경 제공

2) Transaction Management (지출 입력 및 관리)
• Purpose: 사용자의 소비 내역을 정확하게 기록하고 자산을 추적
• Approach: 날짜, 금액, 결제 수단, 메모를 직접 입력하거나 기존 내역을 수정/삭제할 수 있는 UI 제공
• Dynamics: 사용자가 지출이 발생했을 때 또는 과거 내역을 정리하고 싶을 때 발생
• Goals: 데이터의 무결성 유지 및 사용자의 사용 편의성 극대화

3) Data Import / Export (데이터 가져오기 및 내보내기)
• Purpose: 외부 금융 데이터와의 연동성을 높이고 데이터 활용의 자유도를 제공
• Approach: 카드사/은행에서 제공하는 CSV 파일을 업로드하여 대량 입력하거나, 전체 내역을 Excel 파일로 추출
• Dynamics: 타 서비스에서 데이터를 옮겨오거나 별도의 외부 문서 작업이 필요할 때 사용
• Goals: 입력의 자동화 및 데이터 이식성 확보

4) Automatic Category Classification (카테고리 분류)
• Purpose: 수많은 소비 데이터를 체계적으로 관리하여 분석의 기초를 마련
• Approach: 키워드 기반의 알고리즘(예: '식당' 단어 포함 시 '식비' 분류)을 통해 자동 분류하고, 사용자가 수동으로 수정 가능하게 함
• Dynamics: 데이터가 입력되거나 CSV 임포트가 완료된 직후 시스템에 의해 수행
• Goals: 데이터 분류의 정확도 향상 및 사용자 편의성

5) Expense Analysis (지출 패턴 분석)
• Purpose: 단순 기록을 넘어 사용자가 '어디에, 왜' 돈을 썼는지 인사이트를 제공
• Approach: Analysis Engine이 기간별/카테고리별 비중을 계산하고, 이전 달 대비 변동폭을 수치화
• Dynamics: 사용자가 '분석' 탭에 진입하거나 주간/월간 리포트가 생성될 때 동작
• Goals: 소비 증가의 원인을 명확히 규명하고 지출 패턴 파악

6) Budget Setting & Alert (예산 설정 및 알림)
• Purpose: 계획적인 소비를 유도하고 과소비를 사전에 방지
• Approach: 사용자가 설정한 목표 예산과 실시간 지출액을 비교하여 특정 비율(예: 80%, 100%) 초과 시 푸시 알림 발송
• Dynamics: 지출 데이터가 추가되어 예산 임계치에 도달할 때 실시간으로 발생
• Goals: 예산 준수율 제고 및 사용자의 즉각적인 행동 변화 유도

7) Statistics Visualization (통계 시각화)
• Purpose: 재정 상태를 한눈에 파악할 수 있도록 직관적인 정보를 제공함
• Approach: 막대 그래프(추이), 히트맵(요일별 소비량) 등 다양한 시각화 도구 활용
• Dynamics: 사용자가 통계 페이지를 조회할 때 최신 데이터를 바탕으로 렌더링
• Goals: 복잡한 데이터를 가독성 높은 정보로 변환하여 사용자 이해도 증진

8) FAQ & Feedback (문의 및 피드백)
• Purpose: 분석 결과에 대한 궁금증을 해소하고 사용자의 의견을 수렴하여 서비스를 개선
• Approach: 게시판 형태의 Q&A 인터페이스를 통해 질문을 등록하면 관리자나 AI 분석 엔진이 답변을 제공
• Dynamics: 분석 결과가 모호하거나 시스템 개선 제안이 있을 때 사용자가 진입
• Goals: 서비스 신뢰도 구축 및 사용자 요구사항 기반의 지속적인 시스템 업데이트

## 5. Problem Statement

본 시스템은 개인의 소비 데이터를 체계적으로 관리하고 분석하여 사용자의 지출 패턴과 증가 원인을 설명하고 소비를 개선할 수 있도록 지원하는 것을 목표로 한다.

이를 위해 다음과 같은 문제를 해결해야 한다.

### Problem #1 - 분석 결과 왜곡
자동 분석 과정에서 잘못된 데이터 처리로 인해 결과가 왜곡될 수 있다.

**해결 방안**
- 사용자 수정 기능 제공  
- 키워드 기반 분류 룰 개선  

---

### Problem #2 - 카테고리 분류 오류
잘못된 분류는 분석 결과의 신뢰성을 저하시킨다.

**해결 방안**
- 분석 근거 정보 제공  
- 분류 기준의 투명성 확보  

---

### Problem #3 - 개인정보 및 보안 문제
금융 데이터는 민감 정보이므로 보안이 중요하다.

**해결 방안**
- 최소한의 데이터만 저장  
- 민감 정보 마스킹 처리  

---

### Problem #4 - 인사이트 실효성 부족
분석 결과가 실제 행동 변화로 이어지지 않을 수 있다.

**해결 방안**
- 구체적인 피드백 제공  
- 실행 가능한 행동 가이드 포함
